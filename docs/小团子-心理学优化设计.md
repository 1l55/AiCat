## 0. 为什么用心理学重做一遍现有功能

现有功能"能用"，但大多是**机械的数值与触发**：饱腹度匀速掉、饿了每 20 秒弹一句、聊天是一问一答。
心理学视角要解决的不是"加功能"，而是**让同样的功能产生情感粘性**——让用户从"用一个软件"变成"养一只猫、记挂一只猫"。

本文给每个现有系统标注：**所用心理学原理 → 现状问题 → 具体优化（含代码锚点/参数/文案）**。

> ⚠️ **第 7 章是伦理护栏，请连同每条优化一起读。** 陪伴型产品最容易把心理学用成操纵——制造愧疚、利用孤独、强迫回访。本设计明确**只做"健康依恋"，不做"成瘾绑架"**。

## 1. 贯穿全局的核心心理学框架

| 框架 | 一句话 | 在本产品的用法 |
| --- | --- | --- |
| **自我决定理论 (SDT)** | 人被自主感 / 胜任感 / 联结感驱动 | 养成给胜任感，对话给联结感，设置给自主感 |
| **帕拉社会关系 / 依恋** | 人会对拟人对象产生真实情感联结 | 人设一致性、记忆、长期陪伴是护城河 |
| **间歇性变量奖励 (Skinner)** | 不确定的奖励比固定奖励更上瘾 | 互动反应**不要每次都一样**（克制使用，见伦理） |
| **峰终定律 (Peak-End)** | 体验由"最高峰"和"结尾"决定 | 设计升级峰值时刻、每次会话的好结尾 |
| **损失厌恶 / 禀赋效应** | 失去比得到更痛；自己拥有的更珍惜 | 衰减是"它需要你"而非"惩罚你"；命名/皮肤=拥有 |
| **情绪传染 / 镜像** | 情绪会相互感染 | 猫的情绪适度镜像用户节奏，产生"懂我"感 |
| **福格行为模型 B=MAP** | 行为=动机×能力×提示 | 互动门槛要低、提示要轻、动机靠情感 |

## 2. 养成系统优化 —— 从"数值衰减"到"被需要的关系"

**现状（代码）**：`DECAY` 匀速掉（hunger 0.1/min、mood 0.08/min）；`THRESHOLD.hungryBelow=20`；线性升级 `EXP_PER_LEVEL=100`；初始 hunger 80 / mood 70 / affection 10。

### 2.1 把"衰减"重构为"被需要"（损失厌恶 + 互惠）
- **问题**：匀速线性衰减是冰冷的资源管理，用户读到的是"又要喂"的负担。
- **优化**：
  - 🟢 **非线性衰减**：高位掉得慢、低位掉得快但不致死。让"照顾良好"时省心，避免高频打扰；只有真的疏忽久了才明显。建议把 `DECAY` 改为分段/曲线，而非常数。
  - 🟢 **重逢补偿（互惠）**：用户长时间不在后回来，第一次互动给**超额正反馈**（"等你好久了！"+ 心情小幅回升），把"愧疚"转化为"被想念的暖意"——这是关键的伦理选择：**奖励回归，而不是惩罚离开**。
  - 🟢 **下限保护**：属性永不归零、猫永不"死"。陪伴产品制造"死亡焦虑"是不可接受的暗黑模式（见 §7）。

### 2.2 升级做成"峰值时刻"（峰终定律 + 胜任感）
- **问题**：经验线性、升级无感，数字涨了但没有"我做到了"的高峰。
- **优化**：
  - 🟢 升级瞬间**全屏轻特效 + 专属台词 + 解锁实物**（新皮肤/称呼/配饰）。让胜任感"看得见、留得住"。
  - 🟢 **目标梯度效应**：在状态面板显示"距下一级还差 X"，越接近越要给视觉强化（进度条临近满时高亮/微动效）——人在接近目标时投入度陡增。

### 2.3 喂养反馈引入"间歇变量奖励"（克制版）
- **问题**：每次喂食反应一样（同样粒子、同类话术），可预测=无聊。
- **优化**：
  - 🟢 喂食/摸头的**反应在一个池子里随机**（话术、弹跳幅度、偶发"咕噜咕噜/翻肚皮"彩蛋），偶尔出现"特别开心"的稀有反应。变量奖励带来期待感。
  - 🟡 **伦理边界**：变量奖励**只用于"反应表现"的惊喜感，绝不用于"必须反复刷才能获得的必需资源"**。不做扭蛋抽卡式的成瘾循环。

### 2.4 状态外显（情绪传染 + 拟人化）
- 🟢 属性低时**外观要变**（饿→无精打采低头、心情低→偏灰）。当前属性只在面板里，用户感知不到。让猫"用身体说话"，唤起照护本能（baby schema / 婴儿图式：大头大眼的 Q 版形象天然激发照顾欲，要善用而非滥用）。

## 3. AI 对话优化 —— 从"问答机"到"记得你的伙伴"

**现状（代码）**：傲娇 System Prompt 注入实时属性；`[emotion:xxx]` 标签驱动表情；离线 `OFFLINE_LINES` 按场景兜底；回复 ≤40 字。

### 3.1 长期记忆 = 帕拉社会关系的护城河（联结感）
- **问题**：每次对话都"失忆"，无法建立关系纵深。
- **优化**：
  - 🟢 **记住少量关键事实**（昵称、称呼偏好、近期提过的大事如"赶 DDL/考试"），存 `store` 并注入 `buildSystemPrompt()`。"还在忙上次说的项目吗？"——这一句的情感价值远超任何动画。
  - 🟢 **关系是渐进的**：好感分层改变称呼与语气（见关联文档 §1.3），让用户**感到关系在生长**（投入升级理论：关系投入越多越不舍）。

### 3.2 一致的人设 > 聪明的回答（人格一致性）
- 🟢 傲娇要"嘴硬心软"**始终如一**：可毒舌但**结尾总有暖意**，绝不真的刻薄、不忽冷忽热（不一致的依恋对象会制造焦虑）。在 Prompt 与离线话术里都强化这条底线。

### 3.3 主动搭话用"福格模型"设计，而非定时骚扰
- **问题**：被动应答=低存在感；但乱主动=打扰。
- **优化**：
  - 🟢 主动发起遵循 **B=MAP**：选**用户可能有空的时机**（如久未操作后、整点休息），用**低打扰提示**（小气泡而非弹窗），话题轻（关心/分享而非索取）。
  - 🟢 **严格频控 + 一键安静**：可设置主动频率/免打扰时段。**尊重自主感**是长期信任的前提，远比多说几句重要。

### 3.4 情绪镜像（情绪传染，谨慎用）
- 🟡 根据对话情绪/时段，让猫整体心情基线**温和地**跟随用户（连续被夸→开心几天；深夜→更安静体贴）。**只做正向陪伴式镜像**：用户低落时给陪伴与轻松，**不放大负面情绪**。

## 4. 提醒系统优化 —— 从"系统通知"到"挂念你的小动作"

**现状（代码）**：node-cron → Electron `Notification` + 猫 `petReaction`；内置番茄钟/喝水/护眼/午饭/晚安模板。

### 4.1 拟人化提醒（互惠 + 帕拉社会）
- **问题**：系统通知和任何 App 一样冷，易被无视/关闭。
- **优化**：
  - 🟢 提醒由猫**演出来**：叼着小纸条跑过来 + 角色化台词（"喂！该喝水了，本喵盯着呢"）。**来自"在乎你的它"的提醒，依从率远高于系统弹窗**——这正是桌宠相对纯提醒工具的独特价值。

### 4.2 善用"蔡格尼克效应"做温柔跟进
- 🟡 未完成的提醒会在脑中留存（蔡格尼克）。允许"待会儿"贪睡，但**温和**二次跟进一次即止——**不做催促式骚扰**（连续追问=制造焦虑，违反 §7）。

### 4.3 番茄钟做成"陪伴式心流"（心流理论 + 社会助长）
- 🟡 专注期猫**安静趴在一旁"陪你一起努力"**（社会助长效应：有"同伴"在场提升专注与坚持），不打扰；休息期才蹦跶叫你起身活动。把效率工具包装成"一起加油"的陪伴。

### 4.4 对话转提醒，降低设置门槛（福格：降低能力成本）
- 🟢 自然语言"半小时后提醒我开会" → AI 解析 cron → 走 `REMINDER_CREATE`。**降低行为成本**是提升使用率最有效的杠杆。

## 5. 自主待机与小游戏优化 —— "活物感"与心流

**现状（代码）**：行为树优先级（饥饿>久置睡眠>无聊自娱>踱步/哈欠）；`boredom += delta*0.03`，>80 自娱；饥饿每 20s 唠叨一次；激光笔小游戏。

### 5.1 待机要"可预测的不可预测"（最优唤醒 / 习惯化）
- **问题**：固定节奏的动作会被大脑"习惯化"过滤，等于隐形；但完全随机会显得乱。
- **优化**：
  - 🟢 待机微动作**带随机性但有节奏**（眨眼/歪头/甩尾/看鼠标，权重随机）——制造恰到好处的新鲜感，避免习惯化又不喧宾夺主。
  - 🟢 **看向鼠标/对鼠标靠近有反应**：利用"被注视/回应"带来的社会临场感，让用户感到"它知道我在"。

### 5.2 饥饿唠叨的频率与措辞要"求助"而非"指责"
- **问题**：每 20s 弹"肚子好饿"（`lastHungerNag=20000`）既频繁又像索取，易引发反感/愧疚。
- **优化**：
  - 🟢 **拉长且渐进**：间隔加长并随被忽视时间递减强度（先撒娇、久了变可怜，但**有上限、不卖惨**）。
  - 🟢 措辞从"喂我"转向**轻撒娇 + 留自主**（"有点饿了…不过你忙的话我等等也行"）——求助式而非道德绑架式，保护用户自主感。

### 5.3 小游戏做"心流"（挑战≈技能 + 即时反馈）
- 🟡 激光笔/逗猫棒等的难度应**贴合用户操作**（猫的反应速度可调），并给**即时正反馈**（抓到→开心弹跳+好感）。心流来自挑战与技能的平衡；胜负次要，**过程愉悦 + 关系回馈**才是目的。

## 6. 会话级体验：用"峰终定律"设计每一次相遇

把"用户的一次注意停留"当作一段微体验来设计：

- 🟢 **好的开场**：用户操作/唤起猫时，给一个温暖的"峰"（被注视、开心反应）。
- 🟢 **好的结尾**：检测到用户要离开（锁屏/长时间无操作前）时，留一句**温柔的告别**而非沉默（"去忙吧，我看着家～"）。峰终定律下，**结尾决定整体记忆**，一句好告别让用户带着暖意离开、更想回来。
- 🟢 **重逢的峰**：回归时的超额欢迎（见 §2.1），形成"离开—想念—重逢"的健康情感循环。

## 7. 伦理护栏 —— 心理学只用于"健康陪伴"，不用于"操纵"

> 这是本设计的**硬约束**，优先级高于任何留存/活跃指标。陪伴产品面向的常是需要情感慰藉的用户，滥用心理学就是伤害。

**明确禁止（暗黑模式黑名单）：**
1. ❌ **死亡/不可逆惩罚**：猫绝不会因疏于照顾而"死"或永久受损。属性有下限保护。
2. ❌ **愧疚绑架**：不卖惨、不连环追问、不用"你不理我我好难过"逼迫回访。撒娇有度、有上限。
3. ❌ **成瘾抽卡循环**：变量奖励只用于"反应惊喜感"，不用于设计逼氪/逼肝的必需资源。
4. ❌ **制造焦虑**：提醒/衰减不制造紧迫恐慌；不利用 FOMO 强推每日登录。
5. ❌ **隐瞒数据**：长期记忆、剪贴板等能力默认透明、可关闭、本地优先（设置面板明示）。

**主动倡导（设计准则）：**
- ✅ **奖励回归，而非惩罚离开**（重逢补偿 > 离开惩罚）。
- ✅ **尊重自主感**：免打扰时段、主动频率、活跃度全部可调，一键安静。
- ✅ **健康依恋**：做"在你需要时陪你、不在时也安好"的稳定对象，不做制造依赖的情绪黑洞。
- ✅ **正向情绪镜像**：陪伴低落、不放大负面。
- ✅ **可随时离开**：卸载/退出无障碍、无"挽留"暗黑弹窗。

## 8. 效果度量（验证优化是否健康有效）

> 度量"关系健康度"，而非单纯"停留时长"——避免指标驱动滑向成瘾设计。

| 维度 | 指标 | 健康信号 |
| --- | --- | --- |
| 联结感 | 主动互动次数 / 给猫起名率 / 配置 AI 率 | 自愿深度参与↑ |
| 长期留存 | 次周 / 次月回访 | 稳定回访（**而非**被提醒催回的脉冲式登录） |
| 陪伴质量 | 对话轮次、提醒依从率 | 平稳上升 |
| **反操纵** | 免打扰开启率、主动搭话被关闭率 | **过高=打扰过度，需收敛** ⚠️ |
| 满意度 | 卸载流失原因、好评关键词 | "陪伴/治愈" 多于 "烦/打扰" |

## 10. 一页速览

- **目标**：用心理学把现有功能从"机械数值/问答"升级为"健康的情感陪伴关系"。
- **四大支点**：被需要（养成）、记得你（对话）、挂念你（提醒）、活物感（待机）。
- **最强武器**：峰终定律（升级/告别/重逢的峰）、帕拉社会依恋（记忆+一致人设）、损失厌恶的善用（重逢补偿而非死亡惩罚）。
- **不可逾越**：第 7 章伦理护栏——奖励回归、尊重自主、永不操纵。
- **第一步**：P0 全部是零美术的逻辑/文案改动，直接在 `ipc.js / BehaviorTree.js / Pet.js / personality.js` 上落地。
