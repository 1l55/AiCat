> 文档性质：产品设计（PRD + 视觉设定）
> 适用版本：v1.0 现状之上的 v1.x ~ v2.0 演进
> 撰写视角：AI 产品经理
> 最后更新：2026-06-26

## 0. 怎么读这份文档

每个设计点都标注了 **落地成本**，方便排期：

| 标记 | 含义 |
| --- | --- |
| 🟢 **现成** | 用现有 3 套精灵帧（idle/walk/run）+ 现有特效（tint / 粒子 / 弹跳 / 文字）即可实现，**不需要新美术** |
| 🟡 **小美术** | 需要新增少量像素帧或贴图（如 1~2 套表情帧、几个道具图标） |
| 🔴 **大美术** | 需要成套新动画/皮肤，建议外包或 AI 生成后修帧 |

> 技术对接前提（来自 `CLAUDE.md` / 代码现状）：
> - 渲染是 **全屏透明、置顶、点击穿透** 的 PixiJS 覆盖层，所有视觉都靠 `NEAREST` 缩放保持像素锐利。
> - 新增一套动画 = 加一张精灵图 → 在 `assets/sprites/cat-meta.json` 与 `Pet.js#SHEETS` 登记 → 在 `applyState()` 里映射。
> - AI 情绪经 `[emotion:xxx]` 标签回传，渲染层据此切换表情/动作；离线时 `personality.js#offlineReply` 兜底。
> - 任何新交互入口都要走 `constants.js → preload.js → ipc.js` 三处联动。

## 1. 产品定位与人设

### 1.1 一句话定位
**"一只住在你桌面右下角、嘴硬心软的像素猫，会闹脾气、会关心你、陪你把一天过完。"**

它不是工具，是**情感陪伴体**；养成、提醒、对话都是"让关系变深"的手段，而不是功能堆砌。

### 1.2 人设三要素（沿用并强化现有"傲娇"设定）

| 维度 | 设定 | 表现锚点 |
| --- | --- | --- |
| **性格** | 傲娇 / 粘人 / 偶尔毒舌但不刻薄 | 被冷落会闹别扭、被摸头嘴硬身体诚实 |
| **关系** | 把用户当"主人"也当"同伴"，越互动越黏 | 好感度驱动称呼与语气变化 |
| **底线** | 永远不真正伤人、不卖惨绑架 | 毒舌点到为止，闹脾气有上限 |

### 1.3 成长型人设（**新增**，🟢 文案即可落地）
好感度 / 等级不只是数字，应**改变它说话的方式**——这是养成系统最低成本、最高感知的回报。

| 阶段 | 好感度 | 称呼 | 语气样例 |
| --- | --- | --- | --- |
| 陌生 | 0–20 | "喂"/"那个人" | "哼，谁要理你。" |
| 熟悉 | 21–50 | "你" | "……勉强让你摸一下。" |
| 亲密 | 51–80 | "主人" | "今天也辛苦啦，喵～" |
| 羁绊 | 81–100 | 用户昵称 | "等你一整天了，就想见你。" |

> 实现：把阶段注入 `buildSystemPrompt()`，离线话术库 `OFFLINE_LINES` 也按阶段分层。

## 2. 形象设计

### 2.1 体型与比例
- **二头身 Q 版**（现素材即是），亲和力优先；像素分辨率维持 36×30（idle/walk）/ 40×34（run）。
- 缩放档位沿用 `SCALE_PRESETS`（small 3× / medium 4× / large 6×），**建议新增 `tiny:2`** 给小屏/多猫场景。
- 锚点在脚底中心（现状如此），保证落地、拖拽自然。

### 2.2 配色系统（**核心：把"颜色样子"做成皮肤系统**）

现状是单只棕橘猫。建议把"颜色"产品化为 **皮肤（Skin）系统**——这是最高性价比的"形象丰富化"，因为：
- 🟢 **同一套精灵帧 + 不同 `tint` 染色** 就能产出多只"花色不同"的猫，**零新增美术**即可上线第一批。
- 🟡 真正的花纹（奶牛猫/三花/狸花）需要重绘贴图，作为后续付费/成就皮肤。

#### 第一批：纯染色皮肤（🟢 现成，改 `sprite.tint` 即可）

| 皮肤名 | 主色 Hex | tint 值 | 解锁方式 |
| --- | --- | --- | --- |
| 奶橘（默认） | `#E8A05A` | `0xFFFFFF`（原色） | 默认 |
| 奶盐白 | `#F2E9DE` | `0xF2E9DE` | 默认 |
| 墨灰 | `#6E7378` | `0x9AA0A6` | 等级 ≥ 5 |
| 樱花粉 | `#F7B5C4` | `0xF7B5C4` | 好感 ≥ 50 |
| 薄荷绿 | `#A8E0C8` | `0xA8E0C8` | 累计陪伴 7 天 |
| 暗夜黑 | `#3A3A44` | `0x5A5A6E` | 等级 ≥ 10 |

> ⚠️ tint 是**乘法染色**，只能压暗/偏色、不能提亮。浅色皮肤（白/粉/薄荷）需保证原图够亮才不发灰；若原素材偏暗，浅色皮肤需走 🟡 重绘贴图路线。

#### 第二批：花纹皮肤（🔴 大美术，重绘精灵图）
狸花 / 三花 / 奶牛 / 布偶 / 黑白奶牛 —— 每套需重画 idle/walk/run 三张图，登记到 `cat-meta.json`。作为**成就/节日/付费**奖励，制造收集动机。

#### 状态色（沿用现有 tint 语义，明确成规范）

| 状态 | tint | 含义 |
| --- | --- | --- |
| 正常 | `0xFFFFFF` | 默认 |
| 睡眠 | `0x9AA4C8` | 偏冷变暗（现状） |
| 生病/低状态 | `0xB8C0B0`（建议新增） | 灰绿、无精打采 |
| 受惊/生气 | `0xFFC0C0`（建议新增） | 泛红 |

### 2.3 配饰系统（🟡 小美术，叠加贴图层）
在 `Pet.container` 里加一个 **accessory 子精灵**（跟随头部锚点），用小贴图实现"换装"，不动主帧：
- 头部：蝴蝶结 / 圣诞帽 / 草帽 / 耳机
- 颈部：铃铛 / 围巾
- 节日：南瓜灯（万圣）、红包（春节）

> 实现要点：配饰图也走 `NEAREST`，按 facing 翻转，y 跟随呼吸 bob。是养成/节日运营的轻量抓手。

## 3. 待机与动作设计

### 3.1 设计原则
桌宠的灵魂在**待机**——用户 90% 时间是在余光里瞥见它。**绝不能长时间一动不动**。当前 idle 只有 1 帧 + 呼吸 bob，偏单薄，这是第一优先要补的体验。

### 3.2 待机微动作库（Idle Micro-actions）

空闲时由行为树按权重随机播放（在 `BehaviorTree.decide()` 扩展），每个动作结束回到基础呼吸：

| 微动作 | 落地成本 | 实现方式 |
| --- | --- | --- |
| 呼吸起伏（现状） | 🟢 | `sin` bob，已有 |
| 甩尾巴 | 🟡 | 尾巴独立 2 帧贴图循环；或整体轻微 skew |
| 眨眼 | 🟡 | 新增 1 帧"闭眼"，每 4~8s 闪一下 |
| 打哈欠 | 🟢→🟡 | 现用气泡"（打了个哈欠）"占位；🟡 加 2 帧张嘴帧更佳 |
| 舔爪 / 理毛 | 🔴 | 需 3~4 帧专用动画 |
| 歪头好奇 | 🟢 | 整体轻微旋转 ±8°，配 `[emotion:curious]` |
| 坐下/趴下 | 🔴 | 需新姿态帧 |
| 追自己尾巴（自娱） | 🟢 | 复用 run + 原地小圈位移（`performTrick` 已有雏形） |
| 看向鼠标 | 🟢 | 已有 hover 检测，扩展为"鼠标靠近时朝向鼠标 + 歪头" |

> **最低成本高回报组合（建议先做）**：🟢 眨眼节奏 + 歪头 + 看向鼠标 + 甩尾。仅用旋转/位移/单帧叠加，就能让"死站"变"活物"。

### 3.3 时间感知待机（**新增亮点**，🟢 文案+染色）
让猫"知道现在几点"，制造真实陪伴感（读系统时间即可，零成本高感知）：

| 时段 | 行为 |
| --- | --- |
| 清晨 6–9 | 伸懒腰 + "早安喵～" |
| 工作 9–18 | 正常活动，间歇提醒喝水/护眼 |
| 傍晚 18–22 | 更黏人，主动搭话 |
| 深夜 23–次日5 | 睡眠态为主（变暗+Zzz），偶尔迷糊抬头："这么晚还不睡？" |

### 3.4 情绪 → 表现映射（统一规范）
现有 `Emotion` 6 种，建立**表情·动作·特效·音效**四位一体映射表，让 `[emotion:xxx]` 一标签驱动全身反应：

| emotion | 动画 | 特效 | 气泡边框（现状） | 音效（🟡 待补） |
| --- | --- | --- | --- | --- |
| happy | idle/run | happyBounce 弹跳 | 粉 | "喵～"上扬 |
| sad | idle 慢 | 低头（轻旋转） | 蓝 | 低落"呜…" |
| hungry | idle | 食物粒子 spawnFood | 默认 | 肚子咕咕 |
| sleepy | idle 极慢 | Zzz + 冷色 tint | 蓝 | 哈欠 |
| angry | idle | 泛红 tint + 抖动 | 红 | "哈！" |
| curious | idle | 歪头 ±8° | 默认 | "喵？"疑问 |

> 🟡 **建议补 1 套"表情帧"图集**（喜/怒/困/好奇 4 张脸贴在头部位置），比纯靠 tint/气泡表达更生动，是性价比很高的一次美术投入。

## 4. 功能设计

按"陪伴关系"主线组织，而非功能罗列。每项标注优先级（P0 必做 / P1 重要 / P2 增值）与落地成本。

### 4.1 养成系统进化（P0）

现状：饱腹/心情/好感/经验/等级 + 自然衰减 + 喂食/玩耍。问题是**反馈薄、目标弱**。

- **🟢 P0 成长可视化**：等级提升时全屏轻特效（粒子+"Lv Up!"）+ 解锁新皮肤/称呼，让数字"有用"。
- **🟢 P0 状态联动外观**：饱腹<20 → 无精打采（灰绿 tint + 低头待机）；心情高 → 待机更活泼。让属性"看得见"。
- **🟡 P1 喂食多样化**：从单一"喂食"扩展为小鱼干/猫罐头/零食，不同食物加不同属性，复用 `spawnFood` 换粒子颜色。
- **🟡 P1 每日签到 / 亲密任务**："今天摸了它 3 次""陪它玩了 1 局"——轻量日任务驱动回访。
- **🟡 P2 健康度/清洁度** 等新属性（按需，避免数值过载）。

### 4.2 AI 对话进化（P0，这是"AI 桌宠"的核心差异点）

现状：OpenAI 兼容 + 傲娇 System Prompt + 离线兜底 + 情绪标签。已经不错，向"记得你、懂你"演进：

- **🟢 P0 长期记忆**：把用户昵称、偏好、重要事件存进 `store`，注入 System Prompt。"记得你上次说要赶 DDL"——这是情感陪伴的护城河。
- **🟢 P0 主动搭话**：基于时段/久未互动/属性，由主进程定时**主动冒泡**一句话（而非只被动应答）。注意频控，避免打扰。
- **🟡 P1 情绪记忆**：根据近期对话调整整体心情基线（连续被夸→开心几天）。
- **🟡 P1 语音化（TTS）**：接兼容 TTS，给"喵言喵语"配音；先做拟声词音效（🟡）再上完整 TTS。
- **🟡 P2 多模态**："看"剪贴板截图并吐槽（需用户显式授权，隐私优先）。
- **🟢 P0 隐私说明**：设置面板明确"对话是否上云、记忆存在本地"，建立信任。

### 4.3 陪伴与效率（P1）

桌宠的"有用性"来源——和提醒系统结合：

- **🟢 P0 提醒拟人化**：现有 cron 提醒触发时让猫**演**出来（叼着纸条跑过来 + 台词），而非冷冰冰系统通知。
- **🟡 P1 番茄钟伴学**：专注期间猫"陪你一起努力"（趴桌待机），休息期蹦跶叫你起来活动。复用现有番茄钟模板。
- **🟡 P1 自然语言加提醒**：对话里说"半小时后提醒我开会"→ AI 解析为 cron → 走 `REMINDER_CREATE`。把对话和提醒打通。
- **🟢 P2 状态播报**：右键/对话问"今天怎么样"，汇报陪伴时长、互动次数。

### 4.4 小游戏扩展（P1，复用 `MiniGameBase`）

现有激光笔追逐。沿扩展接口加：

| 游戏 | 成本 | 玩法 |
| --- | --- | --- |
| 🟢 逗猫棒 | 🟢 | 拖动鼠标，猫跟随扑（复用 run + 朝鼠标） |
| 🟢 喂食小游戏 | 🟢 | 抛食物，猫接住，复用 spawnFood 反向 |
| 🟡 抓老鼠 | 🟡 | 小老鼠贴图乱跑，点击/猫去抓 |
| 🟡 叠叠乐/猜拳 | 🟡 | 轻量回合制，赢了加好感 |

> 原则：小游戏是**互动加深**手段，胜负次要，都应回馈心情/好感。

### 4.5 收集与运营（P1/P2）

- **🟢 P1 成就系统**："第一次喂食""连续陪伴 7 天""深夜党"——成就解锁皮肤/配饰，给长期目标。
- **🟡 P1 节日皮肤/配饰**：春节红包、万圣南瓜帽、圣诞帽。运营抓手，配饰层（§2.3）低成本实现。
- **🔴 P2 多猫/猫舍**：养第二只、互动，长期方向。

### 4.6 个性化设置（P1）
- 🟢 待机活跃度（安静/正常/活泼）滑块——尊重不同用户对"动来动去"的容忍度。
- 🟢 是否允许主动搭话 + 频率。
- 🟢 多显示器选择猫住在哪块屏。
- 🟡 自定义猫名已支持，扩展为自定义昵称（猫如何称呼你）。

## 5. 优先级路线图

> 原则：先用 🟢 零美术成本把"活物感 + 陪伴感"做透，再投美术做皮肤/表情扩展收集深度。

### Sprint 1 · 让它"活"起来（全 🟢，无新美术）
- 待机微动作：眨眼节奏 + 歪头 + 看向鼠标 + 甩尾（§3.2）
- 时间感知待机（§3.3）
- 纯染色皮肤第一批 + 解锁逻辑（§2.2）
- 状态联动外观：饿/困/开心改变 tint 与待机（§4.1）
- 升级特效 + 解锁反馈（§4.1）

### Sprint 2 · 让它"懂你"（🟢 为主）
- AI 长期记忆 + 主动搭话 + 隐私说明（§4.2）
- 提醒拟人化 + 对话转提醒（§4.3）
- 成长型称呼/语气分层（§1.3）

### Sprint 3 · 让它"耐玩"（🟡 投美术）
- 1 套表情帧图集（§3.4）
- 配饰系统 + 节日皮肤（§2.3 / §4.5）
- 成就系统 + 新增小游戏（§4.4 / §4.5）

### Sprint 4+ · 长期
- 花纹皮肤（🔴）、TTS 语音、多猫猫舍

## 6. 设计约束与技术对接清单

落地时务必对齐（来自代码现状）：

1. **新动画三步登记**：精灵图 → `cat-meta.json` + `Pet.js#SHEETS` → `applyState()` 映射。
2. **tint 只能压暗不能提亮**——浅色皮肤需验证原图亮度，否则改走重绘。
3. **配饰/表情**作为 `Pet.container` 子精灵叠加，跟随 facing 翻转与呼吸 bob，统一 `NEAREST`。
4. **新交互入口**：`constants.js`（IPC + 默认数据）→ `preload.js`（白名单，且 IPC 名需手动同步）→ `ipc.js`（handler）。
5. **新属性/皮肤/成就**加进 `DEFAULT_DATA`，`store.js` 会 deep-merge 保证旧存档兼容。
6. **主动搭话/时段行为**由主进程定时驱动（参考 `scheduler.js` / ipc 衰减循环），经 IPC 通知渲染层，注意**频控防打扰**。
7. **情绪表达**始终经 `[emotion:xxx]` 标签，离线话术库 `OFFLINE_LINES` 要与新场景同步扩充。
8. **性能**：透明置顶覆盖层常驻，待机动作用轻量变换（旋转/位移/单帧/少量粒子）即可，避免高频重绘耗电。
9. **隐私**：长期记忆、剪贴板读取等能力默认关闭或显式授权，本地存储优先。

## 7. 一页速览（给设计/研发对齐用）

- **它是谁**：傲娇粘人的桌面像素猫，情感陪伴 > 工具。
- **形象怎么丰富**：把"颜色"做成**皮肤系统**——先纯染色（零美术），再花纹（重绘）；配饰层做换装与节日运营。
- **待机怎么活**：眨眼/歪头/甩尾/看鼠标 + 时间感知 + 情绪四位一体映射，**先做零美术那批**。
- **功能主线**：活物感 → 懂你（记忆+主动+拟人提醒）→ 耐玩（表情/配饰/成就/小游戏）。
- **第一步**：Sprint 1 全部 🟢，不等美术就能显著提升体验。

## 8. 已落地实现（v1.x，零美术）

> 本章记录截至 2026-06-26 已写入代码的「成就→形象」体系，作为 §2.2 皮肤系统、§2.3 配饰系统、§4.5 成就系统的具体落地。全部 🟢 零新增美术：皮肤用 `tint` 染色，配饰用 `PIXI.Graphics` 矢量绘制。

### 8.1 成就系统（17 个，单一事实来源）
成就目录定义在 `constants.js#ACHIEVEMENTS`，每条 `{ key, name, desc, skin?, accessory? }`，主进程解锁逻辑与渲染层面板共用同一份。解锁走 `ipc.js#tryUnlock(key)`：记录成就 → 连带解锁并**自动装备**对应皮肤/配饰 → 返回通知体；`main.js#announceAchievements()` 逐条弹气泡 + `celebrate()` 粒子，并提示解锁的皮肤/配饰。

| 成就 | 条件 | 奖励皮肤 | 奖励配饰 |
| --- | --- | --- | --- |
| firstFeed | 首次喂食 | 奶盐白 | — |

### 8.2 皮肤系统（16 款，纯染色）
- 目录 `constants.js#SKINS`，每款 `{ key, name, tint, achievement, desc }`；`default` 为初始。
- 渲染（`Pet.js`）：皮肤 `tint` 是**底色**，状态/心情色调（睡眠 `0x9aa4c8`、饥饿 `0xB8C0B0`、低落 `0xC8C8D8`）经 `blendTint()` **叠乘在其上**——睡着的粉猫仍同时偏粉且变冷暗。所有 `sprite.tint` 写入统一走 `_tint()` / `setSkin()`。
- ⚠️ 沿用 §2.2 警告：`tint` 乘法染色只能压暗/偏色，浅色皮肤（奶盐白/樱花粉）若发灰需走重绘（🔴）。

### 8.3 配饰系统（9 件，矢量绘制，4 槽位）
- 目录 `constants.js#ACCESSORIES`，槽位 `ACC_SLOTS = ['head','body','feet','back']`，**同槽互斥、跨槽可叠戴**。
- 绘制（`Pet.js#ACC_DRAW`）：每件配饰一个绘制函数，坐标系为「精灵本地像素」（原点=脚底中心，头顶约 y=-29），用 `PIXI.Graphics` 画纯色块/多边形。背部翅膀画在精灵**之后**的 `wingsLayer`，头/身/脚画在精灵**之前**的 `accessoryLayer`；两层每帧经 `_syncAccessoryTransform()` 跟随精灵的缩放/朝向翻转/呼吸 bob。
- 装备态持久化：`unlockedAccessories`（已解锁）+ `equippedAccessories`（`{slot:key}`，卸下即删键）。

### 8.4 成就 & 形象面板（`ui/AchievementPanel.js`）
「🏆 成就 & 形象」面板三段：成就进度计数 + **皮肤衣橱**（点已解锁皮肤即时换色）+ **配饰衣橱**（按槽位分组，每槽含「无」可卸下）+ 成就墙（显示每条解锁的皮肤/配饰）。切换经 `SKIN_SET` / `ACCESSORY_SET` IPC，主进程 `pushStatus()` 回推让渲染层即时重绘。右键菜单与托盘均可进入。

### 8.5 右键菜单升级为弧形菜单（`ui/MenuPanel.js`）
由竖直列表改为**以点击点为圆心、上半圆扇形展开**的图标按钮（`.pet-radial`），交错弹入动画 + 悬停显示中文标签，圆心自动夹边防出界。8 项：聊天/喂食/玩耍/状态/成就形象/提醒/设置/退出。

### 8.6 旧存档兼容（回填）
`store.js#reconcileUnlocks()`（`init()` 时跑）：皮肤/配饰系统上线**之前**已达成的成就，其对应皮肤/配饰不会凭空出现——回填逻辑遍历已解锁成就，把缺失的 `skin`/`accessory` 补进 `unlockedSkins`/`unlockedAccessories`。配合 `store.js` 的 deep-merge，旧 `petdata.json` 无缝升级。

### 8.7 IPC 契约新增
`ACHIEVEMENTS_GET`（返回 `{achievements, activeSkin, unlockedSkins, equippedAccessories, unlockedAccessories}`）、`SKIN_SET`、`ACCESSORY_SET`。三处联动已对齐：`constants.js#IPC` → `preload.js` 白名单 → `ipc.js` handler。
