## 已完成的功能

### ✅ P0-6: 人设一致性底线 + 渐进称呼

**文件**: `src/shared/personality.js`

**改动内容**:
1. 新增 `affectionTier()` 函数，根据好感度返回称呼与语气：
   - 0-20：疏离戒备，称呼"喂"
   - 21-50：逐渐熟络，称呼"你"
   - 51-80：亲昵粘人，称呼"主人"
   - 81-100：深度羁绊，称呼"主人"（可扩展为用户昵称）

2. 更新 `buildSystemPrompt()`：
   - 注入当前关系阶段（称呼与语气态度）
   - 添加人设底线规则："无论多毒舌，结尾必须流露在乎（嘴硬心软）"
   - 添加伦理红线："不卖惨、不用'你不理我我会难过'逼迫用户"

3. 扩展 `OFFLINE_LINES` 话术库：
   - 为每条话术添加 `minAffection` 字段，实现分层
   - 新增 `reunion`（重逢）和 `farewell`（告别）场景
   - 优化 `hungry` 话术为求助式而非索取式
   - 所有话术遵循"结尾有暖意"的底线

4. 优化 `pick()` 和 `offlineReply()` 函数，根据好感度过滤合适话术

**验收**: 不同好感度阶段称呼与语气明显不同，所有回复（含毒舌）结尾都有暖意

## 心理学原理落地情况

| 心理学原理 | 实施位置 | 效果 |
| --- | --- | --- |
| **损失厌恶 + 互惠** | P0-1 重逢补偿 + 下限保护 | 奖励回归而非惩罚离开，属性永不归零 |
| **自我决定理论（自主感）** | P0-2 求助式文案 | 不道德绑架，给用户自主选择空间 |
| **峰终定律** | P0-4 升级峰值 + P0-5 告别/重逢 | 升级是高峰时刻，告别是好结尾，重逢是新开场 |
| **帕拉社会关系 + 依恋投入** | P0-6 渐进称呼 | 关系随互动加深，称呼语气渐变，产生"关系在生长"感 |
| **人格一致性** | P0-6 人设底线 | 嘴硬心软始终如一，结尾总有暖意，建立稳定依恋 |
| **情绪传染 + 婴儿图式** | P0-3 状态外显 | 用身体语言（色调、呼吸）表达状态，唤起照护本能 |

## 伦理护栏落地情况

✅ **明确禁止的暗黑模式均已避免**:
1. ❌ 死亡/不可逆惩罚 → ✅ 属性有下限保护（P0-1）
2. ❌ 愧疚绑架 → ✅ 求助式而非道德绑架式文案（P0-2），撒娇有度
3. ❌ 成瘾抽卡循环 → ✅ 本次实施无抽卡机制
4. ❌ 制造焦虑 → ✅ 饥饿提示拉长且渐进（P0-2），告别温柔不催促（P0-5）
5. ❌ 隐瞒数据 → ✅ 本次未涉及隐私数据，后续长期记忆需透明可关

✅ **主动倡导的设计准则均已落地**:
- ✅ 奖励回归，而非惩罚离开（P0-1 重逢补偿）
- ✅ 尊重自主感（P0-2 求助式，P0-6 不卖惨）
- ✅ 健康依恋（P0-6 人设一致性，结尾总有暖意）
- ✅ 正向情绪镜像（P0-3 外显属性，陪伴而非放大负面）
