## 1. 衰减下限保护 + 重逢补偿

**目标**：把"属性匀速掉到 0 的惩罚感"改成"它需要你、也想念你"的温暖循环。**奖励回归，而非惩罚离开。**

### 现状代码（`ipc.js:21-32`）
```js
decayTimer = setInterval(() => {
  const pet = store.getPet();
  const hunger = Math.max(0, pet.hunger - DECAY.hunger);      // 会掉到 0
  const moodDecay = DECAY.mood * (hunger < 20 ? 1.6 : 1);
  const mood = Math.max(0, pet.mood - moodDecay);             // 会掉到 0
  store.setPet({ hunger, mood });
  pushStatus();
}, 60 * 1000);
```
问题：①无下限，饿/心情可归零，制造"快死了"焦虑（违反伦理护栏 §7）；②衰减只在 App 开启时跑，但没有"离开多久"的概念，回归时缺少正反馈。

### 改法
**(a) 加下限保护**（改 `ipc.js` 衰减循环）
```js
const FLOOR = { hunger: 15, mood: 20 };   // 建议加进 constants.js
const hunger = Math.max(FLOOR.hunger, pet.hunger - DECAY.hunger);
const mood   = Math.max(FLOOR.mood,   pet.mood - moodDecay);
```
> 永不归零、猫永不"死"。属性低=无精打采，而非濒死。

**(b) 重逢补偿**（新增"离开时长"感知）
- `DEFAULT_DATA.pet` 已有 `updatedAt`；确保每次 `store.setPet` 刷新它（或新增 `lastSeenAt`，在 `STATE_GET` / 退出时写入）。
- 在 `boot()` 取到 `state.pet` 后计算离开时长，分级补偿（**只加不减**）：

```js
// main.js boot()，伪码
const awayMs = Date.now() - (currentPet.lastSeenAt || currentPet.updatedAt || Date.now());
const awayH = awayMs / 3.6e6;
if (awayH >= 2) {
  // 久别重逢：小幅回补心情 + 触发欢迎峰值（见 §5）
  api.updateStatus({ mood: Math.min(100, currentPet.mood + (awayH >= 12 ? 12 : 6)) });
  reunion = true;   // 供 greetLine / 特效使用
}
```

### 验收
- 长时间挂机/重开，饱腹与心情不低于 15/20。
- 关闭 App 数小时后重开，首次问候是"想念式"暖句 + 心情可见回升，而非"我快饿死了"。

## 2. 饥饿唠叨：拉长频率 + 求助式（非绑架式）文案

**目标**：现状每 20 秒弹"肚子好饿"，频繁且像索取，易招反感与愧疚。改成低频、渐进、撒娇而留自主。

### 现状代码（`BehaviorTree.js:31-37`）
```js
if (status.hunger < THRESHOLD.hungryBelow && !sm.is(PetState.WALK)) {
  if (now - this.lastHungerNag > 20000) {          // 20s 太短
    this.lastHungerNag = now;
    this.ctx.bubble.show('肚子好饿…喂我嘛喵～', 'hungry', 3000);  // 索取式
    this.ctx.pet.happyBounce();
  }
}
```

### 改法
**(a) 间隔拉长且渐进**：基础 90s，被忽视越久反而**降低**频率（懂事而非纠缠），并设总上限。
```js
this.hungerNagCount = this.hungerNagCount || 0;
const interval = Math.min(90000 + this.hungerNagCount * 60000, 240000); // 90s→4min 封顶
if (now - this.lastHungerNag > interval) {
  this.lastHungerNag = now;
  this.hungerNagCount++;
  const line = pickHungerLine(this.hungerNagCount); // 见下
  this.ctx.bubble.show(line.t, line.e, 3000);
}
// 被喂食后在 resetIdleTimers() 里重置 hungerNagCount = 0
```
**(b) 求助式渐进文案**（加进 `personality.js#OFFLINE_LINES.hungry`，按次数升级语气，**始终留自主、有上限、不卖惨**）：
```
第1次：'有点饿了…不过你忙的话，我等等也行～'   [hungry]
第2次：'那个…小鱼干什么的，方便的话…嘿嘿'      [hungry]
第3次+：'肚子在唱歌了喵，想到你就忍一忍'         [hungry]   // 到此为止，不再升级
```

### 验收
- 饥饿提示间隔 ≥90s 且随次数变稀。
- 文案无"必须喂我/你不喂我会怎样"的绑架语气；喂食后计数归零。

## 3. 状态外显：让属性"看得见"

**目标**：属性目前只在面板里。让猫**用身体说话**——饿了蔫、困了暗、开心更活泼，唤起照护本能。

### 现状代码（`Pet.js#applyState`）
只按 **状态**（WALK/SLEEP/...）设 tint，不反映**属性高低**。`reactEmotion`（`main.js:208`）也未处理 `sad`。

### 改法
**(a) 新增按属性染色的方法**（`Pet.js`）
```js
// 在 IDLE/CHAT 等中性状态下，叠加属性色调
applyMood(pet) {
  if (this.currentAnim !== 'idle') return;
  if (pet.hunger < 20)      this.sprite.tint = 0xB8C0B0;  // 蔫·灰绿
  else if (pet.mood < 25)   this.sprite.tint = 0xC8C8D8;  // 低落·偏灰
  else                      this.sprite.tint = 0xFFFFFF;
  // 心情高时呼吸更明显（加大 bob 幅度），心情低时放缓
  this._bobAmp = pet.mood > 70 ? 2.2 : pet.mood < 25 ? 0.8 : 1.5;
}
```
- `update()` 里把硬编码的 `Math.sin(this.bobPhase) * 1.5` 改用 `this._bobAmp`。
- 在 `main.js` 的 `onStatusUpdate` 与主循环里调用 `pet.applyMood(currentPet)`。

**(b) 补全 `reactEmotion` 的 `sad`**（`main.js:208`）：低头/放缓 + 蓝气泡已有，加一个轻微下沉动作即可（非弹跳）。

### 验收
- 饱腹 <20 时猫肉眼可见"蔫"（偏色 + 呼吸变弱）；喂食后恢复。
- `[emotion:sad]` 不再走默认无反应。

## 4. 升级峰值时刻 + 目标梯度进度

**目标**：现状升级只弹一句气泡（`main.js:177/192`），无"我做到了"的高峰。用峰终定律放大升级瞬间，用目标梯度让"接近升级"可见。

### 改法
**(a) 升级峰值**（新增 `Pet.celebrate()`，复用现有 Graphics 粒子能力）
```js
celebrate() {
  this.happyBounce();
  for (let i = 0; i < 16; i++) {              // 向上喷一圈星星/亮点
    const g = new PIXI.Graphics();
    g.beginFill(0xFFE08A).drawRect(0,0,5,5).endFill();
    g.x = this.x; g.y = this.baseY - this.height*0.6;
    const a = Math.random()*Math.PI*2, sp = 1+Math.random()*2;
    g.vx = Math.cos(a)*sp; g.vy = -Math.abs(Math.sin(a)*sp)-1;
    g.life = 50; this.effectLayer.addChild(g); this.particles.push(g);
  }
}
```
> `update()` 的粒子循环需支持 `vx`（现仅 `vy`），小改即可。
- `doFeed` / `doPlay` 里 `if (res.leveledUp)` 分支调用 `pet.celebrate()` + 专属台词（"本喵又变强啦！Lv.X 🎉"）。

**(b) 目标梯度进度**（`StatusPanel`）
- 经验线性 `EXP_PER_LEVEL=100`，显示"距下一级还差 N"。
- 进度条 ≥80% 时高亮/加微动效——**越接近越要强化**，利用目标梯度效应提升临门投入。

### 验收
- 升级瞬间有粒子爆发 + 弹跳 + 专属文案（明显区别于普通互动）。
- 状态面板显示到下一级的剩余经验，临近时进度条高亮。

## 5. 温柔告别 / 重逢欢迎（峰终定律）

**目标**：峰终定律下"结尾"决定整体记忆。给每段相处一个暖结尾、给回归一个暖开场。

### 改法
**(a) 重逢欢迎**：改 `greetLine`（`main.js:221`），接 §1(b) 的 `reunion` 标记。
```js
function greetLine(pet, reunion) {
  if (reunion) return pickReunionLine(pet);       // "等你好久啦，就想见你～" 等
  if (!pet) return '喵～';
  if (pet.hunger < 20) return '主人…我有点饿喵';   // 语气软化
  if (pet.mood < 25)  return '哼，你可算回来了…';
  return '喵～欢迎回来！';
}
```
- 重逢时 `bubble.show` 同步 `pet.celebrate()`（小欢迎峰）。

**(b) 温柔告别**：监听离开信号，留一句告别。
- 简易实现：监听 `window` 的 `blur` / `visibilitychange`（长时间无操作 `sm.idleFor()` 超阈值）→ 低频（每次会话最多 1 次）冒泡"去忙吧，我看着家～"。
- 退出/锁屏前写 `lastSeenAt`（供 §1 重逢计算）。

### 验收
- 久别重开 → 想念式欢迎 + 小特效。
- 长时间离开前 → 一句温柔告别（不打扰、不挽留、每会话至多一次）。

## 6. 人设一致性底线 + 渐进称呼

**目标**：傲娇必须"嘴硬心软"始终如一（不一致的依恋对象制造焦虑）；好感增长要**改变说话方式**，让用户感到关系在生长。

### 改法（全部在 `personality.js`）
**(a) System Prompt 增加硬底线**（`buildSystemPrompt`）
```
回复规则补充：
6. 无论多毒舌，结尾必须流露在乎（嘴硬心软），绝不真正刻薄、不忽冷忽热
7. 不卖惨、不用"你不理我我会难过"逼迫用户，撒娇有度
```
**(b) 渐进称呼**：按 `pet.affection` 分层注入称呼与语气（与设计文档 §1.3 一致）
```js
function affectionTier(a) {
  if (a <= 20) return { call: '喂',   tone: '疏离戒备' };
  if (a <= 50) return { call: '你',   tone: '逐渐熟络' };
  if (a <= 80) return { call: '主人', tone: '亲昵粘人' };
  return            { call: '专属昵称', tone: '深度羁绊' };
}
```
- 注入 Prompt；离线 `OFFLINE_LINES` 也按 tier 分层挑选（`offlineReply` 增加 tier 入参）。

### 验收
- 高好感与低好感时，称呼/语气明显不同。
- 任何回复（含毒舌）结尾都有暖意；离线话术同样遵守。
